# 研究计划1：基于态势增强分层多Agent生成与SMT闭环验证的自然语言任务图构建方法（v4.4 精简版）

---

## 一、研究目标

面向无人机集群对抗场景，研究从自然语言作战指令到结构化、可验证、可执行任务有向无环图（DAG）的自动构建方法。

本研究以指挥员自然语言指令为核心输入，结合敌方态势描述、我方集群能力、环境约束与战术知识，生成结构清晰、语义明确的标准化任务计划；随后采用图—约束联合代码表示（GCJP）构建任务图与形式化约束，并通过SMT/Z3验证任务图的时序一致性、资源可行性、能力匹配与协同关系正确性。

本研究聚焦任务规划层，输出“谁在什么时候去哪里做什么”的抽象任务图、关键依赖关系、调度骨架与执行输入参数，不直接生成连续轨迹或底层飞控指令。

---

## 二、问题定义

### 2.1 输入

1. 指挥员自然语言作战指令；
2. 敌方态势描述；
3. 我方无人机集群能力模型；
4. 环境配置，包括目标区域、防御边界、威胁区、会合点、禁飞区等；
5. 动作模板库，包括侦察、跟踪、压制、拦截、打击、待机、会合等；
6. 战术知识库，包括典型反制模式、任务优先级规则与处置建议；
7. 人机交互确认信息。

### 2.2 输出

1. 标准化自然语言任务计划；
2. 候选反制措施清单；
3. 人机确认后的最终任务计划；
4. 图—约束联合代码表示GCJP；
5. 可验证任务DAG；
6. 时间调度方案、关键路径、资源分配结果与下游执行输入参数。

### 2.3 任务图表示

- 节点表示原子任务事件，包含执行主体、动作类型、目标区域、任务参数、资源需求与执行条件；
- 边表示任务依赖关系，包括顺序、并行、同步、屏障、条件触发和信息传递；
- 约束表示任务执行需满足的时间、资源、能力、协同和抽象物理可行性条件；
- 验证结果用于判断任务图是否满足形式化一致性和语义忠实性要求。

---

## 三、核心挑战

1. **多主体跨句语义理解**：识别多无人机集群、多敌方目标和多任务阶段中的省略、指代与隐含事件。
2. **态势依赖型任务补全**：结合敌方态势描述补全任务目标、触发条件、处置优先级与候选行动。
3. **协同时序关系建模**：统一表达顺序执行、并行行动、同步会合、条件触发、全局屏障和联合行动。
4. **多Agent生成一致性**：保证多个规划Agent输出的节点命名、上下游条件、资源状态和跨段依赖一致。
5. **任务层抽象物理可行性**：在任务规划层考虑飞行时间下界、航程预算、弹药预算、能力匹配和同步可达性。
6. **形式化验证与冲突归因**：在验证失败时定位具体节点、依赖边、资源约束或生成Agent。
7. **人机交互式任务确认**：候选反制措施经指挥员确认后，方可融合进最终任务计划。

---

## 四、方法概述

提出一种态势增强的分层多Agent Code-as-Plan框架，形成如下闭环流程：

```text
指令理解 → 态势摘要 → 候选反制 → 人机确认 → 任务分解 → 代码生成 → 全局拼装 → 形式化验证 → 冲突修复
```

整体思路如下：

1. 对指挥员自然语言指令进行结构化解析，形成初步任务计划；
2. 从敌方态势描述中提取威胁摘要，结合我方条件和战术知识生成候选反制措施；
3. 通过人机交互确认形成最终标准化任务计划；
4. 由指挥Agent完成任务分解和协同时序骨架构造；
5. 多个规划Agent并行生成局部GCJP代码；
6. 整合模块完成局部代码拼装和资源状态推导；
7. 验证修复Agent基于图结构检查、SMT求解和语义反向校验完成验证、归因与修复。

## 五、总体框架

### 5.1 态势增强任务计划规范化层

负责将原始指令、态势描述、我方条件和战术知识转化为最终标准化任务计划。

主要功能：

- 原始指令结构化解析；
- 指令歧义检测；
- 敌方态势摘要提取；
- 候选反制措施生成；
- 人机交互确认；
- 最终标准化任务计划生成。

### 5.2 指挥Agent层

负责全局任务理解和任务分解。

主要功能：

- 识别参与集群；
- 划分任务阶段；
- 生成协同时序骨架；
- 标注跨段依赖；
- 生成规划Agent任务包。

### 5.3 规划Agent层

多个规划Agent并行处理局部任务段。

主要功能：

- 局部任务节点生成；
- 局部依赖关系生成；
- 粗粒度可行性筛查；
- 局部GCJP代码生成；
- 局部约束编码；
- 局部冲突报告。

### 5.4 确定性整合层

负责将多个局部结果拼装为全局任务图。

主要功能：

- 局部代码合并；
- 节点命名规范化；
- 跨段依赖连接；
- 资源状态推导；
- 全局任务图构造；
- 全局约束集合生成。

### 5.5 验证修复层

负责对任务图和约束进行递进式验证。

主要功能：

- Python代码执行验证；
- NetworkX图结构验证；
- SMT/Z3约束验证；
- 语义反向校验；
- 冲突归因；
- 局部修复；
- 人工确认回退。

## 六、关键技术路线

### 6.1 输入建模与任务表示

构建统一输入表示体系：

- 自然语言指令表示；
- 标准化任务计划表示；
- 态势摘要表示；
- 集群能力模型表示；
- 动作模板表示；
- NetworkX任务图表示；
- Z3/SMT约束表示。

### 6.2 原始指令结构化解析

从指挥员指令中抽取：

- 执行主体；
- 任务动作；
- 目标区域；
- 目标对象；
- 时间顺序；
- 并行关系；
- 同步关系；
- 条件触发；
- 会合要求；
- 资源约束；
- 能力需求。

同时生成歧义检测报告，识别主体不明、目标不明、方向指代不明、触发条件缺失、优先级缺失和能力冲突等问题。

### 6.3 态势摘要提取

从敌方态势描述中提取可用于任务计划生成的关键要素：

- 敌方集群编号；
- 子群划分；
- 当前空间位置；
- 运动方向；
- 行为模式；
- 可能意图；
- 威胁目标；
- 预计到达时间；
- 威胁等级；
- 处置优先级；
- 态势发展趋势。

示例：

```json
{
  "enemy_group": "E1",
  "behavior": "rapid_breakthrough",
  "approach_direction": "northeast",
  "threat_target": "P1",
  "estimated_arrival_time": 8,
  "threat_level": "high",
  "priority": 1,
  "possible_intent": "breakthrough"
}
```

### 6.4 候选反制措施生成

基于态势摘要、我方资源状态、环境约束和战术知识库，生成候选反制措施清单。

每个候选措施包含：

- 措施编号；
- 适用态势；
- 反制目标；
- 建议动作；
- 推荐执行主体；
- 触发条件；
- 资源需求；
- 预期效果；
- 风险说明；
- 推荐理由；
- 可验证约束提示。

### 6.5 人机交互确认

系统将候选反制措施提交给指挥员确认，支持：

- 全部采纳；
- 部分采纳；
- 修改后采纳；
- 暂不采纳；
- 重新生成；
- 补充新指令。

确认结果形成最终标准化任务计划，并保留候选措施采纳记录、修改字段和计划版本号。

### 6.6 最终标准化任务计划生成

最终任务计划应满足：

- 执行主体明确；
- 任务动作明确；
- 目标区域明确；
- 目标对象明确；
- 任务顺序明确；
- 同步关系明确；
- 条件触发明确；
- 资源需求明确；
- 能力需求明确；
- 可转化为任务图和SMT约束。

### 6.7 指挥Agent任务分解

指挥Agent接收最终标准化任务计划，完成：

1. 任务阶段识别；
2. 参与集群识别；
3. 局部任务段划分；
4. 协同时序骨架构造；
5. 跨段依赖标注；
6. 规划Agent任务包生成。

### 6.8 规划Agent粗粒度可行性筛查

规划Agent在代码生成前进行低成本预检：

- 动作能力匹配；
- 航程预算检查；
- 弹药或载荷检查；
- 时间窗可行性检查；
- 会合点粗略可达性检查；
- 上下游任务条件兼容性检查；
- 并行任务资源竞争检查。

### 6.9 GCJP局部代码生成

规划Agent将局部任务段转化为图—约束联合代码表示。

GCJP代码包含：

- 任务节点定义；
- 任务边定义；
- 节点属性；
- 边属性；
- 时间约束；
- 资源约束；
- 能力约束；
- 同步约束；
- 条件触发约束；
- 约束来源追踪信息。

示例：

```python
g.add_task(
    task_id="T1",
    actor="scout_team_A",
    action="track",
    target="E1",
    duration_lb=5,
    required_capability=["recon"],
    energy_cost=10
)

g.add_dependency(
    source="T1",
    target="T3",
    relation="condition_trigger",
    condition="E1 enters Z1"
)
```

### 6.10 全局拼装与资源状态推导

整合模块执行确定性拼装：

- 合并局部GCJP代码；
- 统一节点命名；
- 补全跨段依赖；
- 推导能源、弹药、载荷和通信资源状态；
- 构建全局任务图；
- 构建全局约束集合。

### 6.11 SMT/Z3约束编码

构建形式化约束模板库，主要包括：

1. 时间顺序约束：`start_j >= end_i`
2. 持续时间约束：`end_i = start_i + duration_i`
3. 时间窗约束：`earliest_i <= start_i <= latest_i`
4. 同步约束：`abs(start_i - start_j) <= sync_tolerance`
5. 资源约束：`sum(resource_cost(actor, tasks)) <= resource_budget(actor)`
6. 能力匹配约束：`required_capability(task) ⊆ capability(actor)`
7. 条件触发约束：`trigger_condition_i → start_j >= trigger_time_i`
8. 抽象物理可行性约束：`flight_time_lb <= available_time_window`

### 6.12 递进式验证机制

验证修复Agent执行四层验证：

1. **代码执行验证**：检查Python语法、受限API调用、危险操作和运行错误；
2. **图结构验证**：检查DAG合法性、节点覆盖、边合法性、孤立节点和拓扑排序；
3. **SMT统一验证**：检查时序一致性、资源守恒、能力匹配、同步可达性和条件触发一致性；
4. **语义反向校验**：从任务图反向重构任务摘要，并与最终标准化任务计划进行一致性对比。

### 6.13 冲突归因与定向修复

冲突类型包括：

- 时间冲突；
- 资源冲突；
- 能力冲突；
- 同步冲突；
- 依赖关系冲突；
- 任务遗漏；
- 语义偏移；
- 代码执行错误。

修复方式包括：

- 调整时间窗；
- 修改依赖边类型；
- 重分配执行主体；
- 调整资源使用；
- 补充缺失节点；
- 删除冗余节点；
- 重新生成局部GCJP代码；
- 回退指挥Agent重新分解；
- 触发人工确认。

## 八、训练策略

### 8.1 总体策略

采用“共享基座—任务分支特化—验证驱动优化—多Agent协同优化”的训练策略。

### 8.2 共享基座阶段

增强模型对无人机任务语义的基础理解能力，训练内容包括：

- 多主体事件识别；
- 动作类型识别；
- 目标区域识别；
- 时序关系理解；
- 条件触发理解；
- 结构化输出能力；
- 无人机领域术语理解。

### 8.3 任务分支特化阶段

针对不同模块设置训练任务：

- 指令解析模块：自然语言指令到标准化任务计划；
- 态势摘要模块：态势描述到结构化威胁摘要；
- 候选反制模块：态势摘要与我方条件到候选反制措施；
- 指挥Agent：最终任务计划到任务分解与协同时序骨架；
- 规划Agent：局部任务包到局部GCJP代码；
- 验证修复Agent：验证报告到冲突解释、修复建议与局部代码修复。

### 8.4 验证驱动优化阶段

利用自动验证信号构造奖励，包括：

- JSON格式合法性；
- GCJP代码可执行性；
- DAG合法性；
- SMT可满足性；
- 资源约束一致性；
- 能力匹配一致性；
- 语义反向校验一致性；
- 修复后通过率；
- 人机确认一致性。

### 8.5 多Agent联合优化

以统一验证器作为集中式评价器，优化目标包括：

- 指挥Agent分解合理性；
- 规划Agent局部生成正确性；
- 跨Agent输出一致性；
- 整体任务图验证通过率；
- 平均修复轮数；
- 并行生成效率。

## 十一、技术风险与应对

### 11.1 态势描述不稳定

- **风险**：上游态势描述可能存在遗漏、误判或表达不一致。
- **应对**：使用结构化摘要字段约束态势输入，并进行威胁等级、处置优先级和关键目标一致性检查。

### 11.2 候选反制措施过度自由生成

- **风险**：LLM可能生成不符合资源条件或任务规则的建议。
- **应对**：使用固定候选措施模板、动作模板库和资源能力检查进行过滤。

### 11.3 多Agent输出不一致

- **风险**：不同规划Agent可能生成冲突节点或不兼容约束。
- **应对**：采用统一命名规则、任务包边界约束、确定性整合模块和SMT全局验证。

### 11.4 GCJP代码生成错误

- **风险**：代码可能出现语法错误、非法API调用或约束遗漏。
- **应对**：使用受限API、代码沙箱、静态检查和执行验证。

### 11.5 SMT约束不可满足

- **风险**：任务计划可能在时间、资源或能力上不可行。
- **应对**：通过冲突核心定位、局部修复、执行主体重分配和人机确认回退处理。

### 11.6 数据集构建成本较高

- **风险**：完整标注任务图、GCJP代码、态势摘要和人机确认记录成本较高。
- **应对**：采用人工种子样本、模板扩展、强模型辅助生成、自动验证筛选和失败样本回收机制。
